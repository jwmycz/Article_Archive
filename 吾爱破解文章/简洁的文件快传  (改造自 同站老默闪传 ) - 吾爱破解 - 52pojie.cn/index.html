简洁的文件快传，适合局域网文件分享的场景

本人只会python，前端开发算是一个很大的挑战，这个信心--- 是 LLM 给的
这次除了开发 有用的文件分享应用，主要也想验证下 Vibe coding 到底可不可以 （说实话很震撼，大家一定要亲自尝试 agent 开发）

# 开发
项目主体参考 老默闪存版，本版相比 IM 版，修改为 FastAPI 框架（原 flask），重构了前端交互（个人偏好），将设置界面 集成到首页增加一些小 featrue- 加 文件广场功能- 加 音视频预览功能（mp4,mp3）

# 关于 大语言模型 LLM
主要使用的是 mimo-v2-flash, 作为 Agent ，通过vs code 调用，
其次是 deepseek 作为辅助(web)，还有qwen 做小代码修复

mimo的模型评价（完成项目主体，统计(4天左右)：请求748次，token消耗 51 百万）
(#目前免费阶段 到1-20为止)
总体来说，没有特别大的困难，非常意外的好用，但需要克服一些小困难（下面类似的能克服，即可用 agent开发）
- 写命令行测试代码，"python -c xxx" ， 经常会遇到 "gbk"解码问题，反复修改（主要是浪费时间）
- 遇到一次，循环查找正则 （被 插件停止）
- 遇到一次，要copy main.js 到 html文件（似乎当 清尾工作）
- (code) 不同的模块写同样的函数，然后分别调用 （不算bug，但可优化）
- (code) 文件名 需要通过 header传递，用base64加密，，mimo 里面理解流程，但生成代码 会忘记 base64解码（用户提醒 才改）
- debug 时候不会自动分模块

# 用 agent 的技巧 （常规 LLM调用技巧 也正常用）
- 能写测试代码，基本就等价成功，如果不能自动化，也要制造反馈的桥梁
- 尽量一次处理一个 小bug （或者 相关联的几个）
- 想办法增加 自己 review的效率，包括debug的手段（瓶颈在人...）, 好几次都 找 deepseek 去检查问题
- (个人向)不要扣细节，agent 生成代码很快，以它的生成效率最大为中心，在这基础上 再思考优化

def startup_event(app:FastAPI):
    """应用启动时的初始化"""
    logger.info("应用启动中...")

    ensure_directories()
    db, should_close = acquire_db_if_needed()
    try:
        try:
            # 初始化默认配置
            init_default_configs(db)

            # 执行启动清理
            startup_cleanup(db)
        except Exception as e:
            logger.error(f"初始化时出错: {e}")

        # 启动后台清理任务（使用APScheduler）
        try:
            scheduler = BackgroundScheduler()

            def cleanup_job():
                try:
                    count, files = cleanup_expired_and_limit_reached_files(db)
                    if count > 0:
                        logger.info(f"后台清理完成，删除了 {count} 个文件")
                except Exception as e:
                    logger.error(f"后台清理任务出错: {e}")

            # 每10分钟执行一次清理
            scheduler.add_job(
                cleanup_job,
                trigger=IntervalTrigger(minutes=CLEAN_INTERVAL),
                id='cleanup_job',
                replace_existing=True
            )

            scheduler.start()
            logger.info(f"后台清理任务已启动，每{CLEAN_INTERVAL}分钟执行一次")

            # 保存调度器实例到应用状态
            app.state.scheduler = scheduler

        except Exception as e:
            logger.error(f"启动后台任务时出错: {e}")
    finally:
        release_db_if_acquired(db, should_close) #db清理

    logger.info("应用启动完成")

async def shutdown_event(app:FastAPI):

@router.get("/api/receive/{code}", dependencies=[Depends(ip_access_control)])
async def get_file_info_by_code(code: str, db: Session = Depends(get_db)):
    """
    通过取件码获取文件信息（用于接收面板显示）
    """
    file_record = get_file_by_extract_code(code, db)
    if not file_record:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="无效的提取码"
        )

    # 检查文件有效性
    is_valid, error_msg = check_file_validity(file_record, db)
    if not is_valid:
        status_code = status.HTTP_404_NOT_FOUND
        if "过期" in error_msg:
            status_code = status.HTTP_410_GONE
        elif "下载次数" in error_msg:
            status_code = status.HTTP_409_CONFLICT

        raise HTTPException(status_code=status_code, detail=error_msg)

    # 计算剩余信息
    remaining_downloads = file_record.max_downloads - file_record.current_downloads
    remaining_time = file_record.expires_at - datetime.now()
    remaining_hours = int(remaining_time.total_seconds() / 3600)

    if remaining_hours < 0:
        remaining_time_str = "已过期"
    elif remaining_hours < 24:
        remaining_time_str = f"{remaining_hours}小时"
    else:
        remaining_days = remaining_hours // 24
        remaining_time_str = f"{remaining_days}天"

    # 判断文件类型
    filename = file_record.original_filename.lower()
    file_type = 'other'
    if filename.endswith(('.jpg', '.jpeg', '.png', '.gif')):
        file_type = 'image'
    elif filename.endswith(('.pdf', '.doc', '.docx', '.xls', '.xlsx', '.ppt', '.pptx', '.txt')):
        file_type = 'document'
    elif filename.endswith(('.zip', '.rar', '.7z')):
        file_type = 'archive'
    elif filename.endswith(('.mp4', '.avi', '.mov')):
        file_type = 'video'
    elif filename.endswith(('.mp3', '.wav')):
        file_type = 'audio'

    return {
        "success": True,
        "file": {
            "name": file_record.original_filename,
            "size": file_record.file_size,
            "note": file_record.note,
            "remaining_downloads": remaining_downloads if remaining_downloads > 0 else "已用完",
            "remaining_time": remaining_time_str,
            "download_url": f"/d/{code}",
            "preview_url": f"/api/preview/{file_record.id}",
            "type": file_type
        }
    }

这代码是怎么使用？直接复制到哪里？小白，期待有客户端

使用很简单，只要电脑已经先装好 python + 联网  
   
去我的github 仓库（ github.com/fun-tailor/q-share），或者直接 用链接下载 （github.com/fun-tailor/q-share/archive/refs/heads/main.zip）
- 下载 zip 压缩包 ---> 解压到随便一个文件夹 （如 I:\测试\q-share-main）
- 打开命令行，输入 “pip install -r ”  ，然后到那个 "q-share-main" , 将 "requirements.txt" 文件直接拖入命令行，然后 按回车键即可
- 然后继续回到 "q-share-main" 文件夹，，鼠标双击 run_gui.py 来启动服务器（这个是带 GUI窗口的版本,直接 run.py 也是可以），再点 “启动服务”， 看日志说成功即可 (这个)
- 然后去浏览器 打开 “http://localhost:5000/” ， 就能看到界面 （服务器一直运行，就能一直打开，或者今天关，明天打开继续用）

在界面中，直接拖文件上去（比如 "xxx.mp4"），或者点击 加号 +，然后点击上传 （可以选择 "分享到广场"）
要取的时候，取件码 和 删除码 都在同一个地方输入，

- 关于局域网访问，需要手机, 电脑都在同一个 路由器下，然后手机输入 http://[电脑的局域网地址]:5000/  （例如 http://192.168.31.10:5000/  这里面192.168.31.10 就是电脑的局域网地址，去路由器管理界面查询出来 ）
不过本人没做好 移动端适配（还在考虑当中），，目前 ipad 这种大屏幕是可以的，，电脑分享电影，音乐，，ipad 是可以在线预览

其他功能可以自己探索
